# DES audience modes and design-taste calibration

Status: `candidate; human promotion authorized`
Evaluation date: `2026-07-18`  
Candidate version: `0.5.0`  
Promotion authority: explicit human approval in the owning thread  

## Question

Can DES improve design taste without applying one visual point of view to every
surface? The candidate had to distinguish a data-heavy admin tool, a public-data
website, an editorial record, and a marketing site; adapt instruction and proof
to the actual model/harness; and retain a falsifiable shared floor.

The intervention borrowed structural fingerprints and product-local design DNA
from Hallmark, independent style dials from Taste, and provider/harness
separation from adjacent design skills. It rejected theme roulette, universal
marketing constraints, and forced model aesthetics.

## Candidate contract

| Mode | Audience and task | Default expression | Reference task |
|---|---|---|---|
| `operator` | Practiced operator completing repeated consequential work | low variance, compact density, functional motion, interface type, evidence-only imagery | Find the page-entry pattern and inspect its implementation contract in `showcase.html` |
| `public-data` | Public reader comparing evidence without assumed fluency | medium variance, balanced density, explanatory motion, editorial-data type | Filter qualifier loss, reach exact wire-desk values, exercise empty scope and reset in `ui-kits/public-data.html` |
| `editorial` | Reader following a sustained argument or documented record | medium variance, relaxed density, quiet motion, editorial type | Read the claim, inspect its evidence, and continue through `ui-kits/weaver.html` |
| `marketing` | Prospective user judging distinction, credibility, and action | high variance, relaxed density, expressive motion, branded type | Understand how DES differs by audience and reach its usage material in `ui-kits/des-marketing.html` |

The shared floor retained honest proof, semantic status, accessibility,
responsive behavior, reachable states, claim/evidence/action custody, and
tuple-bound receipts. Operator-specific bans on decorative gradients, card
shadows, motion theater, and structural novelty no longer apply globally.

## Mechanical and live evidence

- Node test suite: `33 passed`; the final case records unmeasured gradient
  contrast rather than silently treating it as checked.
- Four DES receipts: operator, public-data, editorial, and marketing each passed
  at `1440×1000`, `768×1024`, and `390×844`.
- Each capture contains a viewport PNG, full-page PNG, accessibility snapshot,
  console/page-error state, overflow and resource checks, computed text contrast,
  dead-affordance detection, and a profile attestation.
- `des-profile` binds the receipt to surface mode, harness adapter, requested and
  served model declarations, model tier, and declared capabilities. The digest
  preserves the declaration; it does not independently identify a provider call.
- The public-data live task completed `all → wire → community-empty → reset`:
  wire scope produced `28 tracked / 9 lost / 2 rows`, the URL changed to
  `scope=wire&period=2026-03`, community scope rendered the honest empty state,
  and reset restored `83 tracked / 42 lost` plus the all-scope URL.

During the evaluation, the receipt tool itself exposed and repaired four defects:

1. keyboard focus modality leaked across browser sessions and failed the first
   wide capture;
2. the focus audit changed viewport scroll position before screenshots;
3. Chromium's implicit missing favicon created irrelevant resource failures; and
4. text contrast and visually dead controls were asserted by the rubric but not
   measured by the receipt.

|---|---|
| Gradient/image-backed text was skipped by computed contrast | repaired: capture records `contrast-unmeasured` as an explicit warning instead of silently treating it as measured |
| No durable judge ledger existed | repaired by this evaluation record |
| Mobile-nav source test was narrower than its name | test name narrowed; live accessibility snapshots and three-width receipts remain the behavioral proof |
| Public-data mobile nav wrapped `Exact values` | repaired with non-wrapping navigation labels |
| Contrast used an undocumented `+0.05` grace | repaired: the normative threshold is now applied directly |

Gradient-backed text remains a manual seeing obligation when the browser receipt
emits `contrast-unmeasured`; expressive permission is not an automated contrast
claim.

## Decision

Accept DES `0.5.0` for human promotion. The promotion claim is scoped to:

- four explicit audience/task modes plus independent style dials;
- safer removal of universal operator aesthetics;
- deterministic, self-declared model/harness profiles;
- stronger viewport/full-page receipts with contrast and affordance evidence;
- four mechanically green reference surfaces; and
- an independently reopened `fix-then-reshoot → candidate` judgment.

This does not prove that every future surface improves. Spindle held-out
evaluations must continue binding outcomes to the exact DES package/profile,
requested and served model, harness, capabilities, task, and reference state.
