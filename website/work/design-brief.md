# Design brief

Private working document. Artoo keeps this file outside `site/`; it is not deployed.

## Reader decision

A product designer, frontend engineer, design-systems maintainer, or agent-tool
author should be able to decide whether DES fits their work, choose the right
surface mode, generate an honest model/harness profile, and run the first
auditable implementation without knowing the Lyra Forge portfolio.

## Headline claim

Strong agent-assisted design starts by selecting the audience and task before
the visual style, then binds implementation evidence to the actual model and
harness that produced it.

## Supported claims

- DES 0.5.0 defines four audience/task modes with a shared accessibility and
  evidence floor.
- Surface mode and style dials are independent; neither is a quality score.
- The included Lyra Forge references exercise distinct production-shaped
  operator, public-data, editorial, and marketing contracts.
- `des-profile` and `des-audit` produce self-declared tuple-bound contracts and
  deterministic browser receipts.
- The July 2026 calibration passed all four references at three widths and was
  independently reopened before reaching candidate status.

## Unsupported claims and counter-reading

- The calibration does not prove DES improves every product, model, or team.
- The model/provider fields are attestations, not independently verified
  provider telemetry.
- Lyra Forge is a worked example, not a required brand, component library, or
  visual style for adopters.
- DES is not presently distributed from the public npm registry; the honest
  package path is GitHub or a skill installer pointed at the repository.
- The strongest counter-reading is that the four modes add vocabulary without
  improving task outcomes. The site must therefore foreground the held-out
  evaluation contract and never substitute visual variety for effectiveness.

## Data vintages and denominators

- DES package and mode contract: version 0.5.0, repository revision at build.
- Calibration: 2026-07-18, four surfaces × three viewports = twelve passing
  captures. This is a mechanical sample, not an adoption rate or benchmark.

## Licit comparisons

- Valid: audience, primary job, default density, variance, motion, typography,
  imagery role, evidence obligation, and representative task.
- Invalid: ranking modes from better to worse, averaging the dials into a
  maturity score, treating a style as a mode, or comparing example metrics as
  if they describe one real dataset.

## Selected forms

- Direct chooser: maps the named audience/job to a starting mode.
- Comparison matrix: keeps categorical mode differences adjacent and exact.
- Live reference frames: prove that the examples are usable HTML, not moodboard
  screenshots.
- Profile builder: turns the semantic choice into a copyable CLI contract.
- Vizier proposed distribution charts. Rejected: there is no measured
  distribution and their axes would invent a false quantitative hierarchy.
  Retained its reader-decision, fair-comparison, counter-reading, and adjacent
  labeling checks.

## Closest DES reference

- Marketing for the entry argument and distinctive first impression.
- Public-data for comparison clarity and provenance.
- Editorial for sustained explanation.
- Operator for the profile builder and exact implementation state.

## Anti-reference

- Generic documentation card garden with a centered gradient hero.
- Four copies of one layout with different colors.
- Fake testimonials, adoption metrics, or implied npm availability.
- An interactive-looking selector that does not change content or URL state.
- Lyra-only jargon before the transferable contract is clear.

## Proof required

- Factual proof: every command, version, file link, example route, and test count
  resolves against the repository at the deployed revision.
- Visual and editorial proof: keyboard flow, active states, reduced motion,
  contrast, responsive recomposition, live examples, and an independent seeing
  pass across home, chooser, profile, practice, and examples.
- Offline and firewall proof: zero CDN/runtime dependencies; Artoo build,
  status, and doctor clean; Pages uploads only `website/site` after the local
  build copies public DES assets into it.
