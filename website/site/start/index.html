01
Install

Choose the layer you need.

DES can guide an agent, supply CSS primitives, or serve as a readable reference. These paths can be combined, but none silently installs the others.

Agent practice

Install the repository’s skill with the open skills CLI. Inside Lyra Forge, the Spindle service composes and binds skills; other teams can use their own skill manager.

CSS package

Install directly from GitHub until a registry release exists. The package exports semantic tokens, components, and three restrained base themes.

npm install github:lavallee/des#main

Current distribution limit

DES is not claiming a public npm release today. GitHub installation is the supported public package path represented here; pin a commit instead of main when reproducibility matters.

02
First pass

A ten-minute orientation.

  1. Name one user and one task.

    “Operations lead decides which failed ingest to retry” is usable. “Improve the dashboard” is not.

  2. Choose the surface mode.

    Use the repeated job and audience—not the preferred color palette—to select operator, public-data, editorial, or marketing.

  3. Name the structural fingerprint.

    Queue plus inspector, overview-to-detail explorer, annotated narrative, or proof-led product argument says more than “modern layout.”

  4. Declare the implementation tuple.

    Generate a profile for the actual harness, requested and served model, tier, and capabilities.

  5. Build one production-shaped state.

    Use honest copy, real interaction, and at least one empty, error, or recovery path the product can reach.

  6. Audit what rendered.

    Capture the representative task at wide, tablet, and mobile sizes. Do not promote from source inspection alone.

03
Profile

Turn the decision into a contract.

This public-data example declares that Codex both renders and sees the result. The digest preserves those declarations; it does not authenticate the provider.

des-profile \
  --mode public-data \
  --harness codex \
  --requested-model 'gpt-5' \
  --served-model 'gpt-5' \
  --model-tier frontier \
  --capability browser \
  --capability visual-input \
  --format markdown

04
Evidence

Capture the task, not the homepage.

des-audit \
  --url http://127.0.0.1:4173/explorer \
  --task 'Compare two districts and open the exact values' \
  --surface 'District evidence explorer' \
  --mode public-data \
  --harness codex \
  --model-tier frontier \
  --requested-model gpt-5 \
  --served-model gpt-5 \
  --capability browser \
  --capability visual-input

The audit checks the three DES viewports, semantics, focus indicators, computed text contrast, overflow, dead affordances, console errors, resource failures, and performance budgets. Image-backed contrast remains a manual seeing obligation.

Promotion boundary

A clean receipt is evidence, not authority. The maximum autonomous verdict is candidate; independent seeing or explicit human acceptance is required before a binding or published change.
