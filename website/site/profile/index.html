02
Interpret

What changes—and what cannot.

Mode

Changes the audience, objective, default style range, closest reference, and mode-specific proof questions.

Model tier

Changes context size, instruction explicitness, reference loading, variation strategy, and the need for external seeing.

Harness

Changes how context is supplied, which tools can be exercised, and how browser or provider evidence is attached.

Capabilities

Set the proof ceiling. A model that cannot render or see must not claim rendered or perceptual correctness.

Never tuned by the profile

The tuple does not assign a visual style to a model. It must not force a theme, motion library, conversion formula, stock imagery, or hidden overflow to compensate for presumed model behavior.

03
Limit

A digest is not provider telemetry.

The profile ID hashes the declarations in this form. It proves that the tuple stayed stable between the profile and its receipt. It does not prove which provider served the call or which capabilities were actually exercised.

When routing, fallback, or provider identity affects promotion, attach a provider or evaluation-runner receipt that corroborates the requested and served model.
